bool addPolyFromEncoding(std::vector<short int> encoding, bool checkFeas = false, bool custom = false, spmat_Vec *custAi = {}, vec_Vec *custbi = {})¶

Computes the equation of the feasibility polyhedron corresponding to the given encoding.

Parameters

  • encoding: A vector of +1 and -1 referring to which equations and variables are taking 0 value.

  • checkFeas: The polyhedron is added after ensuring feasibility, if this is true

  • custom: Should the polyhedra be pushed into a custom vector of polyhedra as opposed to LCP::Ai and LCP::bi

  • custAi: If custom polyhedra vector is used, pointer to vector of LHS constraint matrix

  • custbi: If custom polyhedra vector is used, pointer to vector of RHS of constraints

The computed polyhedron is always pushed into a vector of arma::sp_mat and arma::vec If custom is false, this is the internal attribute of LCP, which are LCP::Ai and LCP::bi. Otherwise, the vectors can be provided as arguments. true value to checkFeas ensures that the polyhedron is pushed only if it is feasible.

Return

true if successfully added, else false

Warning

Does not entertain 0 in the elements of *encoding. Only +1/-1 are allowed to not encounter undefined behavior. As a result, not meant for high level code. Instead use LCP::addPoliesFromEncoding.

PolyLCP &addPoliesFromEncoding(std::vector<short int> encoding, bool checkFeas = false, bool custom = false, spmat_Vec *custAi = {}, vec_Vec *custbi = {})¶

Computes the equation of the feasibility polyhedron corresponding to the given encoding.

Parameters

  • encoding: A vector of +1, 0 and -1 referring to which equations and variables are taking 0 value.

  • checkFeas: The polyhedron is added after ensuring feasibility, if this is true

  • custom: Should the polyhedra be pushed into a custom vector of polyhedra as opposed to LCP::Ai and LCP::bi

  • custAi: If custom polyhedra vector is used, pointer to vector of LHS constraint matrix

  • custbi: If custom polyhedra vector is used, pointer to vector of RHS of constraints

The computed polyhedron are always pushed into a vector of arma::sp_mat and arma::vec If custom is false, this is the internal attribute of LCP, which are LCP::Ai and LCP::bi. Otherwise, the vectors can be provided as arguments. true value to checkFeas ensures that each polyhedron that is pushed is feasible. not meant for high level code. Instead use LCP::addPoliesFromEncoding.

Note

A value of 0 in *encoding implies that polyhedron corresponding to fixing the corresponding variable as well as the equation become candidates to pushed into the vector. Hence this is preferred over LCP::addPolyFromEncoding for high-level usage.
